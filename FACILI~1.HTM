0:46Wrap-up & Q&A

"오늘 8가지를 봤습니다. 검색·요약부터 이미지·영상, 보고서, 발상, 그리고 나만의 자동화 봇까지. 공통점은 하나 — Gemini는 검색 도구가 아니라 여러분의 업무 파트너입니다. '오늘 만든 브리핑 봇처럼, 지금 자동화하고 싶은 내 업무는 무엇인가?' 그 하나를 이번 주에 만들어 보세요. 방금 본 모든 프롬프트는 이 가이드에 들어 있고, 끝나고 링크로 공유드립니다. 질문 받겠습니다."

마무리 액션. 사용자에게 index.html 링크 배포 + 넥스트 스텝(오늘/이번주/이번달) 안내.

예상 질문 리스트 (Q&A 준비)

교육 중·후 자주 나오는 질문과 진행자용 답변입니다. 정책·용량 등 수치는 사내 기준/공식 문서로 한 번 더 확인하세요.

🔥 우리 2주 데이터로 본 예상 질문 (우선 대응)
출처가 자꾸 틀리거나 엉뚱한 걸 인용해요. 못 믿겠는데요?
대부분 검색 그라운딩(🌐)을 끈 상태에서 답해 생깁니다. 켜야 출처가 붙고 '소스 보기'가 생깁니다. 사내 자료 정확도는 NotebookLM(자료 한정 인용)으로 받으세요.📊 데이터: 불만(Dislike) 1위가 'Bad Citation' 82%, 검색 CTR 0% — 가장 많이 나올 질문
에이전트를 만들었는데 실제로 실행이 안 돼요. 매일 자동 실행은요?
자연어로 입력·처리·출력을 정의하고, 도구(generate_document 등)를 지정한 뒤 스케줄(예: 매일 09:00)을 걸면 자동 실행됩니다.📊 데이터: Action 실행 2주간 0건인데 에이전트는 26개 생성 — 만들고 못 굴린 상태
만든 에이전트가 '내 에이전트'로 잔뜩인데 뭐가 뭔지 모르겠어요.
이름을 붙여 완성·저장해야 재사용됩니다. 이름 없는 미완성은 0의 가치 — 목록에서 정리하고 재사용하세요.📊 데이터: 이름 없는 '내 에이전트' 중복 다수
만든 노트북을 팀과 공유할 수 있나요? 오디오 요약은요?
공유 링크로 팀에 전파할 수 있고, 스튜디오에서 오디오 개요를 생성할 수 있습니다. 둘 다 거의 안 쓰여 강조 가치가 큽니다.📊 데이터: NotebookLM 노트북 71개 생성, 공유 0건·오디오 0건
검색이 잘 안 되는 것 같아요. 사내 자료는 검색이 왜 안 되죠?
웹 검색은 🌐 토글, 사내 자료는 NotebookLM·연동 커넥터로 분리됩니다. 용도에 맞는 경로를 안내하세요.📊 데이터: 검색 성공률 0% — 검색 기능을 의도대로 못 쓰는 사용자 많음
(분석에 밝은 분) 검색 성공률이 0%던데, 이거 문제 아닌가요?
답변 성공률(96%)이 높으면 검색 성공률·CTR은 낮게 나오는 구조입니다(답으로 바로 해결돼 검색결과 클릭이 불필요). 결함이 아니라 '답으로 해결됐다'는 신호입니다.📊 데이터: 답변 96% vs 검색 0% — 대시보드 주석도 동일 설명
이걸 제 실제 업무에 어떻게 더 쓰죠?
오늘 8단계 중 본인 직무에 가까운 1개를 골라 이번 주 실무에 적용하고, 반복 업무는 에이전트로 자동화하도록 안내하세요.📊 데이터: 232시간 절약했지만 활용이 검색·요약에 편중 — 적용 확장이 핵심
접속 · 계정 · 보안
개인 Google 계정으로도 쓸 수 있나요?
아니요. 반드시 회사 조직 계정으로만 접속합니다. 기존 개인 계정과 충돌이 나면 Chrome 시크릿창(Ctrl+Shift+N)으로 접속하세요.
제가 입력한 내용이나 올린 파일이 구글의 AI 학습에 쓰이나요?
엔터프라이즈 환경은 고객 데이터를 기반 모델 학습에 사용하지 않도록 분리·관리됩니다. 다만 정책상 민감·고객정보 입력 자체를 금지합니다. 구체 약관은 사내 보안 기준을 따르세요.
사용 내역이 로그로 남고 감사 대상인가요?
네, 접근·사용 로깅과 보안 통제(VPC-SC 등)가 적용됩니다. 보안 정책 우회 시도는 감사 대상입니다.
우리 부서/직무엔 안 맞는 것 같은데요?
그 자리에서 실제 업무 1개를 받아 즉석 프롬프트로 시연해 보여주는 게 가장 강력한 설득입니다. 검색·요약·문서 초안은 거의 모든 직무에 적용됩니다.
메모리 (Saved Memory)
기억시킨 내용은 어디 저장되고, 지울 수 있나요?
개인 Saved Memory에 저장되며 설정에서 확인·수정·삭제할 수 있습니다. 개인 단위라 동료에게 공유되지 않습니다.
민감한 내 정보를 기억시켜도 되나요?
안 됩니다. 주민번호·고객정보·인증정보 등은 메모리에 저장 금지입니다. 직무·관심사·선호 톤 같은 비민감 정보만 등록하세요.
검색 그라운딩 · 출처
'소스 보기' 버튼이 안 보여요.
검색 그라운딩(🌐)이 OFF면 안 나옵니다. 토글을 켜면 답변에 출처가 붙고 소스 버튼이 생깁니다.
출처가 틀리거나 엉뚱하면요?
AI는 환각(그럴듯한 오답)이 있을 수 있습니다. 그래서 중요한 답은 반드시 '소스 보기'로 원문을 확인하고, 핵심 의사결정은 사람이 확정합니다.
사내 자료도 검색되나요?
웹 검색과 별개로, 사내 자료는 NotebookLM이나 연동된 커넥터(데이터 스토어)를 통해 활용합니다.
파일 첨부
어떤 형식·몇 개까지 올릴 수 있나요?
PDF·DOCX·XLSX·이미지 등 일반 문서 형식을 여러 개 올릴 수 있습니다. 용량·개수 제한은 사내 안내/공식 문서 기준을 확인하세요.
긴 문서도 정확히 요약되나요?
네. 다만 핵심 누락이 걱정되면 "근거 위치도 표시해줘"를 덧붙여 인용을 함께 받으면 검증이 쉽습니다.
이미지(Nano Banana) · 영상(Veo 3)
생성한 이미지·영상을 실제 업무/대외에 써도 되나요?
내부 시안·아이디어 용도로 적합합니다. 대외 사용은 사내 콘텐츠 정책을 따르고, 실존 인물·타사 로고·브랜드 디자인 합성은 저작권·초상권 문제로 금지입니다.
우리 실제 차종으로 만들 수 있나요?
자사 IP·실차 이미지 합성은 별도 검토가 필요합니다. 시연·실습은 가상 차량 콘셉트로 진행하세요.
영상은 얼마나 길고 오래 걸리나요?
Veo 3는 보통 6~10초 클립이며 생성에 1~3분 걸립니다. 음성·효과음도 함께 입혀집니다.
한국어 프롬프트도 되나요?
됩니다. 다만 이미지·영상은 영어 프롬프트가 디테일(렌즈·조명·톤) 제어에 더 유리할 때가 많습니다.
NotebookLM
YouTube 링크가 안 들어가요/답을 못 해요.
공개·자막 있는 영상이어야 하고, 링크 추가 후 처리에 시간이 걸립니다. 미리 추가해 두면 안정적입니다.
일반 채팅과 뭐가 다른가요?
NotebookLM은 '업로드한 자료 안에서만' 답합니다. 외부 정보가 섞이지 않아 '우리 자료 기반' 답이 필요할 때 안전하고, 출처(인용)를 짚어줍니다.
Deep Research · Idea Generation
왜 이렇게 오래 걸리나요(30분~1시간)?
여러 자료를 검색·교차검증해 보고서를 구성하기 때문입니다. 백그라운드로 돌려두고 다른 일을 하면 됩니다.
사내 자료와 웹을 함께 종합할 수 있나요?
네. 첨부 자료 + 외부 최신 정보를 함께 종합해 보고서로 만들 수 있습니다. 결과는 출처로 검증하세요.
Idea Generation은 왜 영문으로 나오나요?
현재 영문 출력 기준입니다. "한국어로 번역해줘" 후속 지시로 변환하면 됩니다.
No-Code Agent
정말 코딩이 필요 없나요?
네. '무엇·언제·어디로'를 자연어로 쓰면 됩니다. 입력·처리·출력 3단계로 나눠 쓰면 더 정확합니다.
generate_document 같은 '도구'가 뭔가요?
에이전트가 작업을 수행할 때 호출하는 액션(문서 생성, 검색 등)입니다. 자연어로 지시하면 적절한 도구를 골라 실행합니다.
매일 아침 자동 실행은 어떻게 하나요?
에이전트에 스케줄(예: 매일 09:00)을 설정하면 정해진 시각에 자동 실행됩니다.
만든 에이전트를 팀과 공유할 수 있나요?
권한 설정에 따라 공유할 수 있습니다. 단, 민감정보를 자동 처리하는 에이전트는 만들지 마세요.
