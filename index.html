Step 1
①첫 메시지 · 나를 기억시키기⏱ 2분
📍 위치 — 채팅 입력창 (Saved Memory에 자동 저장)

한 번만 등록하면 이후 모든 대화에 직무·관심사·선호 톤이 자동 반영됩니다.

YOU ▸ Chat 저는 IT팀의 사용자이고, IT 업무를 담당합니다. IT 및 자동차 시장 트렌드에 관심이 많고, 보고서 형식의 요약된 답을 선호해요. 기억해주세요.
이런 결과가 나와요
  • "기억하겠습니다" 확인 + 이후 답변이 보고서 톤으로 맞춰짐
💡 효능감 — 매번 자기소개를 반복할 필요가 없어집니다.
주의. 주민번호·고객정보·인증정보 등 민감·개인정보는 메모리에 저장 금지.
Step 2
②검색 그라운딩 · 소스 보기⏱ 4분
📍 위치 — 채팅 + 하단 🌐 검색 토글 ON
  1. 🌐 검색 토글 ON — 켜야 '소스 보기'가 생깁니다.
  2. 프롬프트 입력 후 전송 — Step 1의 메모리가 반영돼 IT·자동차 뉴스가 우선 추천됩니다.
  3. 응답 하단 '소스 보기' 클릭 — 근거 원문 확인.
YOU ▸ Chat (🌐 ON) 제가 관심있어 할 만한 최신 뉴스가 있다면 알려주세요.
이런 결과가 나와요
  • 메모리(IT·자동차)에 맞춘 최신 뉴스 + 출처 링크
  • '소스 보기'로 원문 검증 가능
💡 효능감 — 검색·정리에 쓰던 반나절이 1분으로. 게다가 '나에게 맞춘' 뉴스가 옵니다.
연결 포인트. Step 1 → Step 2로 이어 보여주면 "메모리가 실제로 동작한다"가 한눈에 증명됩니다.

Step 7

이런 결과가 나와요

Step 8
⑧No-Code Agent · 일일 뉴스 브리핑 봇⏱ 8분
📍 위치 — 좌측 메뉴 ▸ Agents ▸ 새 에이전트

오늘의 하이라이트. 코딩 없이 자연어로 입력 → 처리 → 출력을 정의하면 매일 자동 실행되는 봇이 만들어집니다.

한 문장 정의 (전체 지시)
YOU ▸ Agent 매일 아침 Google Search를 사용하여 'KGM 신차 출시 및 전기차 신기술'에 대한 어제자 최신 뉴스 3건을 검색해 줘. 그리고 검색된 뉴스의 핵심 내용을 요약하여 문서 생성(generate_document) 도구로 일일 뉴스 브리핑 문서를 만들어줘.
단계별 정의 (입력 · 처리 · 출력)
  1. 입력(Input)
    "Google Search를 사용하여 'KGM 신차 및 전기차 기술 뉴스'에 대한 어제자 최신 뉴스 3건을 검색해 줘."
  2. 처리(Processing)
    "검색된 뉴스의 핵심 내용을 짧게 요약해 줘."
  3. 출력(Output)
    "요약된 내용을 바탕으로 뉴스 브리핑 문서를 생성(generate_document)해 줘."
이런 결과가 나와요
  • 검색 → 요약 → 문서 생성을 자동으로 수행하는 에이전트
  • 이름 붙이고 스케줄을 걸면 매일 아침 브리핑이 자동 생성
💡 효능감 — 'AI를 쓴다'에서 'AI를 관리한다'로. 반복 업무가 내 손을 떠납니다.
주의. 민감정보를 자동 처리하는 에이전트는 만들지 마세요. 비민감 반복 업무로 한정.

✅ DO · 이렇게 쓰세요

  • 출처 확인 습관 — 중요한 답은 '소스 보기'로 원문 확인
  • 쓰면서 다듬기 — 후속 지시로 톤·길이 즉시 보정
  • 동료와 공유 — 좋은 결과·프롬프트는 공유 링크로 전파
  • 맥락을 충분히 — 자세히 설명할수록 결과 품질↑

⛔ DON'T · 절대 하지 마세요

  • 개인 Google 계정 — 반드시 회사 조직 계정으로
  • 개인·민감정보 입력 — 고객정보·주민번호·인증정보 금지
  • 무검토 외부 발송 — 출처·사실 확인 없이 외부로 금지
  • 결과를 '정답'으로 단정 — 핵심 의사결정은 사람이 확정
